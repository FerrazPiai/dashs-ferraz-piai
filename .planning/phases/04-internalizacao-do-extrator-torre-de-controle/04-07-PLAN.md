Script Node ESM que roda fora do servidor (standalone). Invocacao:
```
node scripts/smoke-extractors.js --platform all --user-id 42 --urls urls.json
```

Le `urls.json` com shape:
```json
{
  "transcricao": "https://docs.google.com/document/d/ABC/edit",
  "slides": "https://docs.google.com/presentation/d/DEF/edit",
  "figma": "https://www.figma.com/file/XYZ/Foo",
  "miro": "https://miro.com/app/board/AAA="
}
```

Para cada plataforma filtrada, chama `dispatchExtractor(platform, url, { userId })` e:
- Mede tempo
- Mede `texto.length`, `imagens.length`, `erros.length`
- Gera hash SHA-256 do `texto`
- Salva em `scripts/smoke-output/<timestamp>-<platform>.json` com `{platform, url, ms, texto_len, imagens_len, erros, hash}`

Imprime tabela final com status por plataforma.

Exit code: 0 se todas passaram, 1 se alguma lancou.

Nao requer DB se `INTERNAL_EXTRACTORS` nao incluir plataforma que dependa de user token (ex: figma + miro). Para Google (transcricao/slides), precisa de user_id valido com token Google persistido (ou skippar com warning).
  </action>
