# Phase 4 — UI Design Contract

> Visual and interaction contract for the frontend surfaces of the Torre de Controle extractor internalization (OAuth per-user Google). Generated by gsd-ui-researcher, to be verified by gsd-ui-checker.

**Scope (3 UI surfaces only):**

1. **OAuth Block-on-trigger modal** (D-03) — interceptado no clique de "Analisar projeto" quando o usuario nao tem token Google valido.
2. **Fail-mark reauth banner** (D-04) — aparece no card/drawer de uma analise cujo `status_avaliacao = "incompleta"` por `google_reauth_required`.
3. **Card "Conexao Google"** (D-05) — dentro de uma aba nova "Minha conta" no Painel Admin existente (`/admin`), visivel a todos os roles autenticados.

**Fora de escopo:** novo layout da Torre de Controle, novos dashboards, interfaces para admin gerenciar tokens de terceiros.

## Interaction Contract

### Surface 1 — OAuth modal

| Trigger | Expected behavior |
|---------|-------------------|
| Usuario clica "Conectar Google" | Botao fica disabled, label muda para "Redirecionando...", spinner aparece, `window.location.href = '/api/auth/google/connect-drive?redirect=<path-atual>'` |
| OAuth callback success | Modal fecha automaticamente, trigger original (analise) e disparado com feedback toast "Analisando projeto..." |
| OAuth callback failure | Modal fecha, toast vermelho com copy do error state correspondente |
| Usuario clica "Agora nao" ou clica no backdrop | Modal fecha, analise **nao** e disparada, nenhum estado fica pendente |
| Tecla ESC | Equivalente a "Agora nao" |

### Surface 2 — Reauth banner

| Trigger | Expected behavior |
|---------|-------------------|
| Banner renderiza | Background `rgba(239,68,68,0.08)`, borda esquerda `3px solid #ef4444`, `border-radius: 6px` |
| Usuario clica "Reconectar Google" | Mesma rota do modal (`/api/auth/google/connect-drive?redirect=<path-atual>`), apos callback banner se atualiza e "Retentar analise" fica habilitado |
| Usuario clica "Retentar analise" (habilitado) | `POST /api/torre-controle/analyses/{id}/retry` com spinner no botao, toast "Analise reiniciada" em sucesso |
| Status muda para `active` | Banner muda visual temporario (background `rgba(34,197,94,0.08)`, icone `check-circle-2`) por 2s e depois desaparece |

### Surface 3 — Google connection card

| Trigger | Expected behavior |
|---------|-------------------|
| Tab "Minha conta" abre | `GET /api/google/status` — loading state com spinner inline |
| Status = `active` | Mostra email, datas, badge verde "Ativo", botoes "Desconectar" (secundario vermelho) + "Reconectar" (secundario cinza, util se quiser trocar de conta) |
| Status = `expired` ou `revoked` | Mostra email em cinza, badge vermelho, esconde "Desconectar", destaca "Reconectar" como CTA primario |
| Status = `no_token` | Mostra empty state com CTA primario "Conectar conta Google" |
| Usuario clica "Desconectar" | Abre modal de confirmacao destrutiva (copy acima). Se confirmar: `POST /api/google/disconnect` + atualiza card |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Modal tem foco trap | `VStatusModal` ja teleporta para body; adicionar `@keydown.esc="close"` e `tabindex` no primeiro botao |
| Botao CTA tem aria-label descritivo | `aria-label="Conectar conta Google para autorizar leitura de Docs e Slides"` |
| Banner de reauth tem `role="alert"` | Garante que leitores de tela anunciem a expiracao |
| Contraste de texto >= 4.5:1 | `#fff` em `#141414` = 16.9:1 (ok); `#ccc` em `#141414` = 11.5:1 (ok); `#888` em `#141414` = 5.1:1 (ok como meta label) |
| Icones sempre acompanhados de label | Nunca botao apenas com icone nesta fase (exceto fechar modal, que tem `title="Fechar"`) |
| Tab order segue fluxo visual | Modal: Heading -> Body -> CTA primario -> CTA secundario -> Backdrop |
